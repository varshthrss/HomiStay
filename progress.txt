BUG FIXES (2026-05-27 - Session 2)
-----------------------------------

[Search.jsx - Blank White Page Crash Fix]
- CRITICAL: Search page crashed to blank white screen after map feature was added.
- Root cause: filteredProperties (useMemo) was referenced inside a useEffect BEFORE it was declared.
  useMemo values are not hoisted, so reading them before declaration throws a ReferenceError.
- Fix: Moved filteredProperties useMemo block above the map useEffect that uses it.
- Order is now: state declarations → filteredProperties useMemo → map useEffect → view mode useEffect.

[PropertyDetails.jsx - Booking Card Price Visibility Fix]
- Price breakdown (nights × guests × price, cleaning fee, total) was rendered BELOW the
  "Proceed to Payment" button in Step 2 (Guest Details), forcing users to scroll down to see cost.
- Fix: Moved price breakdown to appear ABOVE the guest detail form fields in Step 2,
  so users immediately see the full cost summary before filling in their details.

[data.sql - Map Showing Only 1 Marker Fix]
- Map on /search page showed only 1 property marker out of 19.
- Root cause: Properties insert used ON CONFLICT (id) DO NOTHING, so pre-existing rows
  kept their original null latitude/longitude values from older DB state.
- Fix: Changed to ON CONFLICT (id) DO UPDATE SET latitude, longitude, city, country, address
  so coordinates are always refreshed on startup even for existing property rows.
- After backend restart, all 19 properties will have valid coordinates and appear on the map.

SECURITY & VALIDATION
---------------------
- All new endpoints are JWT-protected (via Spring Security)
- Host-only endpoints verify property/booking ownership before allowing changes
- Guest modification requests verify the authenticated user owns the booking
- SQL uses safe migrations (ADD COLUMN IF NOT EXISTS, CREATE TABLE IF NOT EXISTS)
- Seed data uses ON CONFLICT DO NOTHING for idempotent inserts
